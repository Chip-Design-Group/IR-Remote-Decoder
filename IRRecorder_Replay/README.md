# IR Recorder & Replay – Mini Universal Remote

This extension adds a **Record & Replay function** for the NEC infrared protocol to the **IR Remote Decoder Project**.

The FPGA can now:
- **Record** received IR codes,
- Store them **in internal Block RAM (BRAM)**,
- and **transmit** them later (Replay).

This turns the pure decoder into an **active mini universal remote control**.

---

## Features

- Recording of **NEC-compatible IR codes**
- Storage in **FPGA-internal Block RAM**
- Replay of stored codes at the push of a button
- Pure hardware approach (no softcore, no external RAM)
- Fully **simulatable and testable** with Cocotb
- Modular design, cleanly integrated into the existing project

---

## Design Philosophy

**No raw pulses or timings** are stored.
Instead, the system stores the **already decoded NEC data**:

| Field | Width |
|---|---|
| Address | 16 Bit |
| Command | 8 Bit |
| Flags (e.g. Repeat/Valid) | 8 Bit |
| **Sum** | **32 Bit per Code** |

Advantages:
- Very low memory usage
- Robust reproduction of IR signals
- Clear separation between decoder, storage, and encoder

## Module Overview

```
IR_RX
↓
NECDecoder
↓
IR_Recorder ──► BRAM (IR_Storage)
↓
Buttons ──► Replay_FSM ──► NEC_Encoder ──► IR_TX ──► NPN-Transistor ──► IR_LED
```

### Module Details

| Module | Description |
|---|---|
| `IR_Recorder` | Receives valid NEC frames and writes them into BRAM |
| `ir_bram` | FPGA-internal Block RAM for storing IR codes |
| `Replay_FSM` | Controls the playback of stored codes |
| `NEC_Encoder` | Generates the NEC protocol (Leader, Bits, Stop) |
| `IR_TX` | 38 kHz carrier generation and modulation for the IR LED via NPN driver stage |

---

## Memory Organization (BRAM)

- Storage as 32-bit words
- Example: 8 memory slots
```
Slot 0: { addr[15:0], cmd[7:0], flags[7:0] }
Slot 1: { ... }
...
Slot 7
```

The BRAM is generated by the synthesizer via inference
(`ram_style = "block"`).

---

## Operating Concept (MVP: 2 Buttons)

| Input | Function |
|---|---|
| REC-Button | Save last valid IR code to Slot 0 |
| PLAY-Button | Play Slot 0 |

For the initial version, only **one slot (Slot 0)** is used.
The concept remains easily extensible (more slots, counters, display).

## Tests

The extension is fully testable with **Cocotb**.

### Recommended Tests

1.  **Record Test**
    - Simulated NEC Frame
    - Record button active
    - Check BRAM content

2.  **Replay Test**
    - Predefined code in BRAM
    - Trigger replay
    - Check timing of the NEC signal

3.  **Closed Loop Test**

    Decode → Store → Replay → Decode

    The re-decoded code must be identical to the original one.

## Prerequisites

- Working NECDecoder
- IR Receiver
- IR LED (38 kHz) with NPN transistor as driver (e.g. BC547 or 2N2222)
- Clock base suitable for µs timing
- Optional: Buttons or switches

### Hardware Note (IR TX Path)

The IR LED should not be driven directly by the FPGA pin, but via an NPN driver stage:

- FPGA `IR_TX` -> Base resistor (e.g. 1 kOhm) -> Base of NPN
- Emitter of NPN -> GND
- Collector of NPN -> Cathode of IR LED
- Anode of IR LED -> Resistor -> VCC

This allows higher LED currents without overloading the FPGA output.
